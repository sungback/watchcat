# Agent Router — 토큰 효율 운영 규칙

## 원칙

복잡도가 낮고 반복적인 작업은 Haiku(`claude-haiku-4-5-20251001`)로 위임하여 토큰 비용을 절감합니다.

## Haiku 위임 대상

| 작업 유형 | 트리거 패턴 | 예시 |
|---|---|---|
| 테스트 실행 요약 | "테스트 돌려줘", "unittest 결과" | 실패 목록 파싱 → 1줄 요약 |
| 로그 분석 | "로그 확인", "에러 찾아줘" | 스택트레이스 → 원인 1줄 |
| 단순 파일 리네임/이동 | "파일 옮겨줘", "이름 바꿔줘" | git mv + 커밋 |
| CHANGELOG 항목 작성 | "릴리스 노트", "변경사항 요약" | git log → 마크다운 bullet |
| pyproject.toml 버전 bump | "버전 올려줘" | 숫자만 바꾸는 Edit |

## Sonnet (현재 모델) 유지 대상

| 작업 유형 | 이유 |
|---|---|
| path_fixer.py 로직 수정 | 충돌/롤백 엣지케이스 판단 필요 |
| CI workflow 디버깅 | Flutter pubspec + serious_python 조합 이해 필요 |
| 새 기능 설계 | 아키텍처 판단 필요 |
| 보안·무결성 관련 변경 | 실수 비용이 높음 |

## 토큰 낭비 방지 행동 룰

| 상황 | 금지 행동 | 대신 할 행동 |
|---|---|---|
| 코드베이스 탐색 시작 | 파일 전체 읽기 | `ls`/`find`로 구조 파악 후 필요한 파일만 읽기 |
| 빌드·테스트 출력 처리 | 전체 로그 컨텍스트 적재 | 에러 코드·실패 줄만 요약 |
| 긴 파일 분석 | `Read` 전체 읽기 | `offset`/`limit`으로 필요 구간만 읽기 |
| 반복 실패 (3회↑) | 같은 접근 반복 | 세션 교체 (`agents/reset-handoff.md`) |
| CI 실패 디버깅 | 앱 코드부터 수정 | `build/flutter/pubspec.yaml` 먼저 확인 |

## 사용법

새 세션 시작 시 작업 유형을 먼저 확인하고,
Haiku 위임 대상이면 모델을 명시적으로 지정하거나 `/model` 명령으로 전환합니다.
